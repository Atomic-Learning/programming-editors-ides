When writing code you will usually edit plain text files: sequences of characters without presentation formatting. These files are typically saved with a file extension that conventionally indicates the programming language — for example .py for Python or .java for Java — though extensions are conventions and can be overridden.

You will often edit these files using a text editor or an Integrated Development Environment (IDE). There are many options, each with its own features and trade-offs. Some tools are multi-language generalists, while others are specialised for particular languages or ecosystems.

Text Editors vs. IDEs

Text editors are applications for creating and editing plain text. At minimum they offer editing, search/replace, and often syntax highlighting. Examples include Notepad++, Sublime Text, Vim, and Emacs. Many modern editors (for example, Visual Studio Code) are highly extensible via plugins or extensions that add features such as language servers, linters, or debuggers.

IDEs provide a more integrated development workflow: an editor plus tools such as a debugger, build or run configurations, an integrated terminal, project-aware navigation and refactoring, test runners, and version-control integration. Examples include Visual Studio, IntelliJ IDEA, and Eclipse. The boundary between editors and IDEs is a spectrum — some editors can be extended into full IDEs, and some IDEs are more lightweight.
